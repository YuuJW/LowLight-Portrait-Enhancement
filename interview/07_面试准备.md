## 1. 面试流程

### 1.1 典型流程

```
简历筛选 → 一面（技术面） → 二面（技术面/主管面） → HR 面 → Offer
```

### 1.2 各轮侧重点

| 轮次 | 面试官 | 侧重点 |
|------|--------|--------|
| 一面 | 技术面试官 | 基础知识、项目细节、编程能力 |
| 二面 | 技术主管 | 项目深度、解决问题能力、技术视野 |
| HR 面 | HR | 职业规划、稳定性、薪资期望 |

### 1.3 时间分配

典型一面（45-60分钟）：
- 自我介绍：3-5 分钟
- 项目介绍：15-20 分钟
- 技术问题：20-25 分钟
- 反问环节：5-10 分钟

## 4. 技术问题

### 4.1 ISP 基础

**Q1: 描述一下 ISP Pipeline 的主要模块**

```
ISP Pipeline 主要分为三个域：

RAW 域：
- 黑电平校正（BLC）：去除传感器暗电流
- 坏点校正（DPC）：修复坏像素
- 镜头阴影校正（LSC）：补偿边缘暗角
- 白平衡增益（WB）：校正色温
- 去马赛克（Demosaic）：Bayer → RGB

RGB 域：
- 色彩校正矩阵（CCM）：映射到标准色彩空间
- Gamma 校正：非线性亮度映射
- 降噪

YUV 域：
- 对比度调整
- 锐化
- 饱和度调整
```

**Q2: Demosaic 算法有哪些？各有什么优缺点？**

```
1. 双线性插值：
   - 优点：简单快速
   - 缺点：边缘伪彩明显

2. 边缘自适应：
   - 优点：根据梯度选择插值方向，减少伪彩
   - 缺点：计算量增加

3. MLRI / VNG：
   - 优点：效果好
   - 缺点：计算复杂

4. 基于学习的方法：
   - 优点：效果最好
   - 缺点：需要大量计算
```

**Q3: AWB 的常用算法有哪些？灰度世界假设有什么局限性？**

```
常用算法：
1. 灰度世界法：假设场景平均颜色为灰色
2. 白点检测法：假设最亮点是白色
3. 灰度边缘法：假设边缘平均颜色为灰色
4. 基于学习的方法：使用神经网络估计色温

灰度世界的局限性：
- 对单一颜色主导的场景失效（如蓝天、绿草地）
- 对严重偏色的场景效果差
- 需要场景中有足够多的颜色分布
```

**Q4: 什么是 CCM？如何标定？**

```
CCM（Color Correction Matrix）是 3x3 色彩校正矩阵，
用于将传感器色彩空间转换到标准色彩空间（如 sRGB）。

标定方法：
1. 拍摄标准色卡（如 X-Rite ColorChecker）
2. 提取色卡上各色块的 RGB 值
3. 获取色块的标准参考值
4. 通过最小二乘法求解矩阵

[R']   [m00 m01 m02] [R]
[G'] = [m10 m11 m12] [G]
[B']   [m20 m21 m22] [B]
```

**Q5: HDR 成像的原理是什么？如何处理鬼影？**

```
原理：
1. 快速连续拍摄多张不同曝光的照片
2. 短曝光保留亮部细节，长曝光保留暗部细节
3. 通过对齐和融合合成高动态范围图像
4. 色调映射转换为可显示的 LDR 图像

鬼影处理：
1. 检测：比较多帧之间的差异，识别运动区域
2. 处理策略：
   - 在运动区域只使用参考帧
   - 降低运动区域的融合权重
   - 使用光流估计运动并补偿
```

### 4.2 AI 相关

**Q6: 讲一下 DnCNN 的网络结构和设计思路**

```
网络结构：
- 采用 VGG 风格：Conv-BN-ReLU 堆叠
- 17 层深度，64 个特征通道
- 不使用池化，保持分辨率

设计思路：
- 残差学习：预测噪声而非干净图像
- 噪声比图像内容简单，更容易学习
- 梯度传播更稳定，训练更快收敛
```

**Q7: 如何将 PyTorch 模型部署到移动端？**

```
步骤：
1. 模型导出：PyTorch → ONNX
2. 模型优化：
   - 量化（FP32 → FP16/INT8）
   - 剪枝（移除不重要的通道）
3. 选择推理框架：
   - ONNX Runtime
   - NCNN（腾讯开源，移动端优化）
   - MNN（阿里开源）
   - SNPE（高通 NPU）
4. 硬件加速：利用 GPU/NPU
```

**Q8: 模型量化的原理是什么？量化会带来什么问题？**

```
原理：
将 FP32 权重和激活值映射到低精度（FP16/INT8）
减少内存占用和计算量

量化方法：
- 训练后量化（PTQ）：直接转换，简单但精度损失大
- 量化感知训练（QAT）：训练时模拟量化，精度损失小

问题：
- 精度下降：特别是 INT8 量化
- 溢出风险：需要合适的缩放因子
- 不是所有层都适合量化
```

**Q9: 你了解 NPU 吗？NPU 和 GPU 有什么区别？**

```
NPU（Neural Processing Unit）是专门为神经网络推理设计的处理器。

与 GPU 的区别：
| 方面 | GPU | NPU |
|------|-----|-----|
| 设计目的 | 通用并行计算 | 神经网络加速 |
| 能效比 | 较低 | 较高 |
| 支持算子 | 灵活 | 固定算子集 |
| 适用场景 | 训练 + 推理 | 主要用于推理 |
| 移动端 | 耗电大 | 省电 |

手机 NPU 例子：高通 Hexagon、华为 Da Vinci、vivo V 系列芯片
```

### 4.3 工程能力

**Q10: 如何优化图像处理算法的性能？**

```
1. 算法层面：
   - 选择更高效的算法
   - 减少不必要的计算

2. 代码层面：
   - 循环展开
   - 避免分支预测失败
   - 内存访问优化（cache友好）

3. 指令层面：
   - SIMD 指令（SSE/AVX/NEON）
   - 并行化（OpenMP）

4. 硬件层面：
   - GPU 加速（CUDA/OpenCL）
   - NPU 加速
```

**Q11: 你了解 SIMD 指令吗？举个例子说明如何使用**

```
SIMD（Single Instruction Multiple Data）可以用一条指令
同时处理多个数据。

例如，使用 NEON 加速 RGB 转灰度：

// 标量版本
for (int i = 0; i < n; i++) {
    gray[i] = 0.299 * r[i] + 0.587 * g[i] + 0.114 * b[i];
}

// NEON 版本（一次处理 8 个像素）
float32x4_t v_r = vld1q_f32(r);
float32x4_t v_g = vld1q_f32(g);
float32x4_t v_b = vld1q_f32(b);
float32x4_t v_gray = vmlaq_n_f32(vmlaq_n_f32(vmulq_n_f32(v_r, 0.299f), v_g, 0.587f), v_b, 0.114f);
vst1q_f32(gray, v_gray);
```

**Q12: C++ 中如何管理内存？有哪些常见的内存问题？**

```
内存管理方式：
1. 栈内存：自动分配和释放
2. 堆内存：new/delete 或 malloc/free
3. 智能指针：unique_ptr、shared_ptr

常见问题：
1. 内存泄漏：忘记 delete
2. 野指针：使用已释放的内存
3. 重复释放：delete 两次
4. 缓冲区溢出：写入超出分配的范围
5. 内存对齐问题：SIMD 要求对齐

解决方案：
- 使用智能指针
- 使用 RAII 原则
- 使用工具检测（Valgrind、AddressSanitizer）
```

**Q14: 多目标跟踪和单目标跟踪有什么区别？**

```
| 方面 | 单目标跟踪（SOT） | 多目标跟踪（MOT） |
|------|-----------------|-----------------|
| 目标数量 | 1 个 | 多个 |
| 初始化 | 第一帧给定目标 | 自动检测所有目标 |
| 核心问题 | 外观建模、遮挡处理 | 数据关联、ID 保持 |
| 典型方法 | SiamFC、TransT | DeepSORT、ByteTrack |
| 评价指标 | AUC、Precision | MOTA、IDF1 |
```
