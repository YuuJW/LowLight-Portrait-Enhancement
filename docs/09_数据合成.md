## 1. 数据问题分析

### 1.1 公开数据集现状

| 数据集 | 类型 | 数量 | 问题 |
|--------|------|------|------|
| LOL | 暗光/正常光配对 | 500 对 | 数量少，景物为主 |

**核心问题**：高质量的"成对"暗光人像数据集极其稀缺。

### 1.2 为什么需要合成数据

```
┌─────────────────────────────────────────────────────────┐
│  问题：需要 (暗光图, 正常光图) 配对数据                  │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  现实拍摄困难：                                          │

│  • 成本高、效率低                                        │
│                                                          │
│  解决方案：数据合成                                      │
│  • 正常光图片 → 模拟暗光退化 → 暗光图片                  │
│  • 配对数据天然对齐                                      │
│  • 可以无限生成                                          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 2. 数据合成原理

### 2.1 暗光图像的特点

| 特点 | 原因 | 模拟方法 |
|------|------|---------|
| **亮度低** | 曝光不足 | 乘以 0.1~0.5 |
| **噪声大** | 高 ISO | 添加泊松噪声 + 高斯噪声 |
| **色偏** | 白平衡错误 | R/B 通道乘以随机系数 |
| **细节丢失** | 低信噪比 | 模糊 + 量化 |

### 2.2 退化模型

**简单退化模型**：
```
暗光图 = (正常图 × 暗化系数) + 噪声 + 色偏
```

**物理退化模型**（更真实）：
```
暗光图 = ISP( RAW_noise( inverse_ISP(正常图) ) )
```

## 3. 物理噪声模型

### 3.1 相机噪声来源

```
┌─────────────────────────────────────────────────────────┐
│  光子 → 传感器 → 电信号 → ADC → 数字值                   │
├─────────────────────────────────────────────────────────┤
│     ↓         ↓          ↓                              │
│  Shot Noise  Read Noise  量化噪声                       │
│  (泊松分布)  (高斯分布)  (通常忽略)                     │
└─────────────────────────────────────────────────────────┘
```

### 3.2 Shot Noise（散粒噪声）

- **来源**：光子到达传感器的随机性
- **特点**：与信号强度相关，暗处更明显
- **分布**：泊松分布

```python
# 泊松噪声
shot_noise = np.random.poisson(signal * k) / k - signal
```

### 3.3 Read Noise（读出噪声）

- **来源**：电路热噪声、放大器噪声
- **特点**：与信号无关，恒定
- **分布**：高斯分布

```python
# 高斯噪声
read_noise = np.random.randn(*signal.shape) * sigma
```

### 3.4 组合噪声模型

```python
# 完整噪声模型
noisy = signal + shot_noise(signal) + read_noise
```

## 4. 代码实现

### 4.1 简单合成（快速实验）

```python
# data_synthesis_simple.py
import numpy as np
import cv2
import random

def synthesize_low_light_simple(img, dark_ratio=None, noise_level=None):
    """
    简单暗光图像合成

    Args:
        img: 输入图像 (H, W, 3)，范围 [0, 255]
        dark_ratio: 暗化系数，默认随机 [0.1, 0.5]
        noise_level: 噪声强度，默认随机 [5, 25]

    Returns:
        noisy_img: 合成的暗光图像
    """
    img = img.astype(np.float32) / 255.0

    # 1. 暗化
    if dark_ratio is None:
        dark_ratio = random.uniform(0.1, 0.5)
    img = img * dark_ratio

    # 2. 添加噪声
    if noise_level is None:
        noise_level = random.uniform(5, 25)
    noise = np.random.randn(*img.shape).astype(np.float32) * (noise_level / 255.0)
    img = img + noise

    # 3. 色偏
    img[:, :, 0] *= random.uniform(0.8, 1.2)  # B
    img[:, :, 2] *= random.uniform(0.8, 1.2)  # R

    # 裁剪
    img = np.clip(img, 0, 1)

    return (img * 255).astype(np.uint8)


if __name__ == '__main__':
    img = cv2.imread('normal_light.jpg')
    noisy = synthesize_low_light_simple(img)
    cv2.imwrite('low_light.jpg', noisy)
```

### 4.2 物理模型合成（更真实）

```python
# data_synthesis_physical.py
import numpy as np
import cv2
import random

class LowLightSynthesizer:
    """
    基于物理模型的暗光图像合成器

    面试话术："由于公开的实拍暗光人像数据稀缺，我构建了一套基于物理模型的
    噪声合成管线，利用 FFHQ 生成了 10,000 对训练数据，不仅解决了数据问题，
    还极大提升了模型的泛化能力。"
    """

    def __init__(self, shot_noise_k=0.1, read_noise_sigma=0.02):
        """
        Args:
            shot_noise_k: 泊松噪声系数
            read_noise_sigma: 读出噪声标准差
        """
        self.shot_noise_k = shot_noise_k
        self.read_noise_sigma = read_noise_sigma

    def inverse_isp(self, img):
        """
        反向 ISP：RGB → RAW 域

        简化版本：逆 Gamma + 逆色彩矩阵
        """
        # 归一化
        img = img.astype(np.float32) / 255.0

        # 逆 Gamma (假设 gamma=2.2)
        img = np.power(img + 1e-8, 2.2)

        # 逆色彩矩阵（简化为直接使用）
        # 实际应该用 CCM 的逆矩阵

        return img

    def add_noise(self, raw, dark_ratio=0.3):
        """
        在 RAW 域添加噪声

        Args:
            raw: RAW 图像（线性空间）
            dark_ratio: 暗化系数
        """
        # 暗化（模拟欠曝）
        raw_dark = raw * dark_ratio

        # Shot Noise（泊松噪声）
        # 泊松分布的方差等于均值
        k = self.shot_noise_k
        shot_noise = np.random.poisson(raw_dark / k + 1e-8) * k - raw_dark

        # Read Noise（高斯噪声）
        read_noise = np.random.randn(*raw_dark.shape) * self.read_noise_sigma

        # 组合
        noisy_raw = raw_dark + shot_noise + read_noise
        noisy_raw = np.clip(noisy_raw, 0, 1)

        return noisy_raw

    def forward_isp(self, raw):
        """
        正向 ISP：RAW → RGB

        简化版本：Gamma 校正
        """
        # Gamma 校正
        img = np.power(raw + 1e-8, 1/2.2)

        # 裁剪
        img = np.clip(img, 0, 1)

        return img

    def add_color_bias(self, img):
        """添加色偏"""
        img[:, :, 0] *= random.uniform(0.85, 1.15)  # B
        img[:, :, 2] *= random.uniform(0.85, 1.15)  # R
        return np.clip(img, 0, 1)

    def synthesize(self, img, dark_ratio=None):
        """
        完整合成流程

        Args:
            img: 输入图像 (H, W, 3)，范围 [0, 255]
            dark_ratio: 暗化系数，默认随机

        Returns:
            noisy_img: 合成的暗光图像，范围 [0, 255]
        """
        if dark_ratio is None:
            dark_ratio = random.uniform(0.1, 0.4)

        # 1. 反向 ISP（RGB → RAW）
        raw = self.inverse_isp(img)

        # 2. 添加噪声（在 RAW 域）
        noisy_raw = self.add_noise(raw, dark_ratio)

        # 3. 正向 ISP（RAW → RGB）
        noisy_img = self.forward_isp(noisy_raw)

        # 4. 添加色偏
        noisy_img = self.add_color_bias(noisy_img)

        return (noisy_img * 255).astype(np.uint8)


def batch_synthesize(input_dir, output_dir, num_samples=10000):
    """
    批量合成数据集
    """
    import os
    from glob import glob
    from tqdm import tqdm

    os.makedirs(output_dir, exist_ok=True)
    os.makedirs(os.path.join(output_dir, 'low'), exist_ok=True)
    os.makedirs(os.path.join(output_dir, 'high'), exist_ok=True)

    synthesizer = LowLightSynthesizer()
    images = glob(os.path.join(input_dir, '*.png')) + glob(os.path.join(input_dir, '*.jpg'))

    for i, img_path in enumerate(tqdm(images[:num_samples])):
        img = cv2.imread(img_path)
        if img is None:
            continue

        noisy = synthesizer.synthesize(img)

        # 保存
        basename = f'{i:05d}.png'
        cv2.imwrite(os.path.join(output_dir, 'low', basename), noisy)
        cv2.imwrite(os.path.join(output_dir, 'high', basename), img)

    print(f'Generated {num_samples} pairs')


if __name__ == '__main__':
    # 单张测试
    synthesizer = LowLightSynthesizer()
    img = cv2.imread('normal_light.jpg')
    noisy = synthesizer.synthesize(img)
    cv2.imwrite('low_light_physical.jpg', noisy)

    # 批量合成
    # batch_synthesize('data/FFHQ/', 'data/FFHQ_lowlight/', num_samples=10000)
```

## 5. Face Mask 预处理

### 5.1 为什么需要离线生成 Mask

```
❌ 错误做法：训练时实时生成 mask
   每个 batch 都运行人脸分割网络 → 极慢

✅ 正确做法：离线预处理
   训练前生成所有 mask 并保存 → 训练时直接加载
```

### 5.3 训练时使用 Mask

```python
# dataset.py
import torch
from torch.utils.data import Dataset
import cv2
import numpy as np
import os

class LowLightFaceDataset(Dataset):
    def __init__(self, data_dir, transform=None):
        self.data_dir = data_dir
        self.low_dir = os.path.join(data_dir, 'low')
        self.high_dir = os.path.join(data_dir, 'high')
        self.mask_dir = os.path.join(data_dir, 'mask')
        self.transform = transform

        self.images = [f for f in os.listdir(self.low_dir) if f.endswith('.png')]

    def __len__(self):
        return len(self.images)

    def __getitem__(self, idx):
        img_name = self.images[idx]

        # 加载图像
        low = cv2.imread(os.path.join(self.low_dir, img_name))
        high = cv2.imread(os.path.join(self.high_dir, img_name))

        # 加载预处理好的 mask
        mask_name = img_name.rsplit('.', 1)[0] + '_mask.npy'
        mask = np.load(os.path.join(self.mask_dir, mask_name))

        # 转换
        low = cv2.cvtColor(low, cv2.COLOR_BGR2RGB).astype(np.float32) / 255.0
        high = cv2.cvtColor(high, cv2.COLOR_BGR2RGB).astype(np.float32) / 255.0

        if self.transform:
            # 同时变换图像和 mask
            pass

        # 转为 tensor
        low = torch.from_numpy(low.transpose(2, 0, 1))
        high = torch.from_numpy(high.transpose(2, 0, 1))
        mask = torch.from_numpy(mask[np.newaxis, ...])  # (1, H, W)

        return low, high, mask
```

## 7. 面试要点

### 高频问题

**Q1: 你的训练数据是怎么来的？**

> 由于公开的暗光人像数据稀缺，我构建了一套基于物理模型的噪声合成管线。
> 使用 FFHQ 高清人脸数据集作为 Ground Truth，通过模拟相机噪声生成配对数据。
> 噪声模型包括 Shot Noise（泊松分布）和 Read Noise（高斯分布），
> 这是工业界处理数据稀缺问题的常规做法。

**Q2: 合成数据和真实数据有什么区别？**

| 方面 | 合成数据 | 真实数据 |
|------|---------|---------|
| 配对性 | 完美配对 | 难以配对 |
| 噪声分布 | 可控 | 复杂多样 |
| 退化类型 | 需要显式建模 | 自然包含 |
| 数量 | 无限 | 有限 |

> 我的做法：合成数据为主（10,000 对），真实数据（LOL）为辅，混合训练。

**Q3: Face Parsing Loss 是怎么实现的？**

> 1. **离线预处理**：训练前用 BiSeNet 为所有图像生成 Face Mask
> 2. **训练时加载**：直接从文件读取 mask，避免重复计算
> 3. **损失计算**：`loss = L1(output, gt) + weight * L1(output * mask, gt * mask)`
>
> 这样既保护了人脸区域，又不影响训练速度。

**Q4: 为什么要在 RAW 域加噪声？**

> 真实相机噪声发生在 RAW 域（传感器输出），不是 RGB 域。
> Shot Noise 的强度与信号强度相关，这个特性在线性 RAW 域才能正确模拟。
> 在 RGB 域加噪声会导致噪声分布不真实，影响模型泛化能力。
