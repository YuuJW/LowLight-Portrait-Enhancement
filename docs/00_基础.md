## 一、C++ 核心基础 (重点中的重点)

### 1.1 内存管理：指针 (`*`) 与引用 (`&`) 的区别

**指针**：存储变量的内存地址，可以为空，可以重新指向其他变量

```cpp
int a = 10;
int b = 20;
int* ptr = &a;    // ptr 指向 a 的地址

std::cout << *ptr << std::endl;  // 输出 10 (解引用)
ptr = &b;         // 指针可以重新指向 b
std::cout << *ptr << std::endl;  // 输出 20

ptr = nullptr;    // 指针可以为空
```

**引用**：变量的别名，必须初始化，不能为空，不能改变引用目标

```cpp
int a = 10;
int b = 20;
int& ref = a;     // ref 是 a 的别名

std::cout << ref << std::endl;  // 输出 10
ref = 30;         // 修改 ref 就是修改 a
std::cout << a << std::endl;    // 输出 30

// int& ref2;     // 错误！引用必须初始化
// ref = &b;      // 错误！不能改变引用目标（这只是把 b 的值赋给 a）
```

**实际应用场景**：

```cpp
// 函数参数：用引用避免拷贝，提升性能
void processImage(const cv::Mat& image) {  // 引用传递，不拷贝图像数据
    // 处理图像...
}

// 函数参数：用指针表示可选参数
void enhance(cv::Mat* output = nullptr) {  // 指针可以为空，表示可选
    if (output != nullptr) {
        // 输出结果...
    }
}
```

**总结对比**：

| 特性 | 指针 | 引用 |
|-----|------|------|
| 可以为空 | ✅ 可以 | ❌ 不可以 |
| 可以重新指向 | ✅ 可以 | ❌ 不可以 |
| 必须初始化 | ❌ 不必须 | ✅ 必须 |
| 访问方式 | `*ptr` 解引用 | 直接使用 |
| 常见用途 | 可选参数、动态内存 | 避免拷贝、别名 |

### 1.2 `new`/`delete` 与智能指针

**传统方式（容易内存泄漏）**：

```cpp
int* arr = new int[100];   // 在堆上分配内存
// ... 使用 arr ...
delete[] arr;              // 必须手动释放，否则内存泄漏！

// 问题：如果中间发生异常，delete 可能不会执行
```

**智能指针（推荐）**：

```cpp
#include <memory>

// std::unique_ptr：独占所有权，不能复制，只能移动
std::unique_ptr<int[]> arr = std::make_unique<int[]>(100);
// 离开作用域自动释放，无需 delete

// std::shared_ptr：共享所有权，引用计数
std::shared_ptr<MyClass> ptr1 = std::make_shared<MyClass>();
std::shared_ptr<MyClass> ptr2 = ptr1;  // ptr1 和 ptr2 共享对象
// 当所有 shared_ptr 都销毁时，对象才被释放
```

**项目中的实际例子**（NCNN 推理）：

```cpp
class NCNNInference {
private:
    std::unique_ptr<ncnn::Net> net_;  // 独占网络对象

public:
    NCNNInference() : net_(std::make_unique<ncnn::Net>()) {}
    // 析构时自动释放，不需要手写 delete
};
```

**智能指针对比**：

| 特性 | `unique_ptr` | `shared_ptr` |
|-----|--------------|--------------|
| 所有权 | 独占 | 共享 |
| 可以复制 | ❌ | ✅ |
| 可以移动 | ✅ | ✅ |
| 开销 | 几乎无 | 引用计数开销 |
| 常见用途 | 类成员、工厂模式 | 多处共享对象 |

### 1.4 移动语义 (`std::move`)

**问题**：大对象拷贝很慢

```cpp
std::vector<float> createLargeVector() {
    std::vector<float> v(1000000);  // 100万个元素
    return v;  // 返回时会拷贝？
}

std::vector<float> result = createLargeVector();  // 传统：拷贝 100万元素
```

**解决方案：移动语义**（C++11）

```cpp
// 移动：转移资源所有权，而不是拷贝
std::vector<float> v1(1000000);
std::vector<float> v2 = std::move(v1);  // v1 的数据"移动"到 v2
// 此时 v1 为空，v2 拥有数据，没有拷贝发生

// 实际应用：函数返回值优化
cv::Mat processImage(cv::Mat input) {
    cv::Mat result;
    // 处理...
    return result;  // 编译器自动优化为移动，不会拷贝整个图像
}
```

**移动 vs 拷贝**：

```
拷贝：
  v1: [1][2][3][4][5]  →  复制数据  →  v2: [1][2][3][4][5]
  v1 仍然有效

移动：
  v1: [1][2][3][4][5]  →  转移指针  →  v2: [1][2][3][4][5]
  v1 变为空（资源被"偷走"）
```

### 1.5 STL 容器：`std::vector`

**基础用法**：

```cpp
#include <vector>

std::vector<int> v;           // 空向量
std::vector<int> v2(10);      // 10 个元素，默认值 0
std::vector<int> v3(10, 5);   // 10 个元素，值都是 5
std::vector<int> v4 = {1, 2, 3, 4, 5};  // 初始化列表

v.push_back(100);             // 末尾添加元素
v.pop_back();                 // 删除末尾元素
v[0] = 10;                    // 访问元素
v.size();                     // 获取大小
v.empty();                    // 是否为空
v.clear();                    // 清空
```

**底层原理与扩容机制**：

```
初始状态：
  capacity = 4, size = 4
  [1][2][3][4]

push_back(5) 时，容量不足，需要扩容：
  1. 分配新内存（通常是原来的 2 倍）：capacity = 8
  2. 拷贝/移动旧元素到新内存
  3. 释放旧内存
  4. 添加新元素

  capacity = 8, size = 5
  [1][2][3][4][5][ ][ ][ ]
```

**性能建议**：

```cpp
// 如果知道大小，预先分配，避免多次扩容
std::vector<cv::Mat> tiles;
tiles.reserve(16);  // 预分配 16 个位置
for (int i = 0; i < 16; i++) {
    tiles.push_back(tile);  // 不会触发扩容
}
```

void worker(int id) {

### 2.2 互斥锁与死锁

**问题：多线程访问共享数据**

```cpp
int counter = 0;

void increment() {
    for (int i = 0; i < 10000; i++) {
        counter++;  // 不安全！多个线程同时修改
    }
}

// 结果可能不是 20000
std::thread t1(increment);
std::thread t2(increment);
```

**解决方案：互斥锁**

```cpp
#include <mutex>

std::mutex mtx;
int counter = 0;

void increment() {
    for (int i = 0; i < 10000; i++) {
        mtx.lock();    // 加锁
        counter++;     // 临界区：同一时刻只有一个线程能执行
        mtx.unlock();  // 解锁
    }
}

// 更安全的写法：lock_guard（RAII）
void increment_safe() {
    for (int i = 0; i < 10000; i++) {
        std::lock_guard<std::mutex> lock(mtx);  // 构造时加锁，析构时自动解锁
        counter++;
    }  // 离开作用域自动解锁，即使抛出异常也不会死锁
}
```

**死锁示例**：

```cpp
std::mutex m1, m2;

void thread1() {
    m1.lock();
    // ... 做一些事情 ...
    m2.lock();  // 等待 m2
    // ...
    m2.unlock();
    m1.unlock();
}

void thread2() {
    m2.lock();
    // ... 做一些事情 ...
    m1.lock();  // 等待 m1 → 死锁！
    // ...
    m1.unlock();
    m2.unlock();
}

// thread1 持有 m1，等待 m2
// thread2 持有 m2，等待 m1
// 两个线程互相等待，永远无法继续 → 死锁
```

**避免死锁**：

```cpp
// 方法1：始终按相同顺序加锁
void thread1() {
    std::lock_guard<std::mutex> lock1(m1);
    std::lock_guard<std::mutex> lock2(m2);
    // ...
}

void thread2() {
    std::lock_guard<std::mutex> lock1(m1);  // 和 thread1 相同顺序
    std::lock_guard<std::mutex> lock2(m2);
    // ...
}

// 方法2：使用 std::scoped_lock (C++17)
void thread_safe() {
    std::scoped_lock lock(m1, m2);  // 同时锁定多个互斥锁，避免死锁
    // ...
}
```

### 2.3 条件变量

```cpp
#include <condition_variable>

std::mutex mtx;
std::condition_variable cv;
bool ready = false;

void worker() {
    std::unique_lock<std::mutex> lock(mtx);
    cv.wait(lock, []{ return ready; });  // 等待直到 ready 为 true
    // 被唤醒后继续执行...
    std::cout << "Worker 开始工作" << std::endl;
}

void master() {
    {
        std::lock_guard<std::mutex> lock(mtx);
        ready = true;
    }
    cv.notify_one();  // 唤醒一个等待的线程
    // cv.notify_all();  // 唤醒所有等待的线程
}
```

**条件变量工作流程**：

```
Worker 线程:
  1. 获取锁
  2. 检查条件 (ready == false)
  3. 条件不满足，释放锁并等待
  4. 被唤醒后重新获取锁
  5. 再次检查条件 (ready == true)
  6. 条件满足，继续执行

Master 线程:
  1. 获取锁
  2. 设置 ready = true
  3. 释放锁
  4. 通知等待的线程
```

### 2.4 生产者-消费者模型

这是你项目中线程池的核心模式：

```cpp
#include <queue>
#include <mutex>
#include <condition_variable>
#include <thread>

template<typename T>
class ThreadSafeQueue {
private:
    std::queue<T> queue_;
    std::mutex mtx_;
    std::condition_variable cv_;

public:
    // 生产者：添加任务
    void push(T item) {
        {
            std::lock_guard<std::mutex> lock(mtx_);
            queue_.push(std::move(item));
        }
        cv_.notify_one();  // 通知消费者
    }

    // 消费者：获取任务（阻塞等待）
    T pop() {
        std::unique_lock<std::mutex> lock(mtx_);
        cv_.wait(lock, [this]{ return !queue_.empty(); });  // 等待非空
        T item = std::move(queue_.front());
        queue_.pop();
        return item;
    }
};

// 使用示例
ThreadSafeQueue<cv::Mat> taskQueue;

// 生产者线程
void producer() {
    for (auto& tile : tiles) {
        taskQueue.push(tile);  // 添加任务
    }
}

// 消费者线程
void consumer() {
    while (true) {
        cv::Mat tile = taskQueue.pop();  // 获取任务（阻塞等待）
        process(tile);
    }
}
```

**生产者-消费者模型图解**：

```
生产者线程                      任务队列                      消费者线程
    │                            │                            │
    ├─── push(task1) ──────────→ [task1] ←───── pop() ────────┤
    ├─── push(task2) ──────────→ [task2] ←───── pop() ────────┤
    ├─── push(task3) ──────────→ [task3] ←───── pop() ────────┤
    │                            │                            │
```

### 2.5 原子操作

```cpp
#include <atomic>

std::atomic<int> counter(0);  // 原子变量

void increment() {
    for (int i = 0; i < 10000; i++) {
        counter++;  // 原子操作，无需加锁
        // 等价于 counter.fetch_add(1);
    }
}

// 常用原子操作
counter.load();           // 读取
counter.store(100);       // 存储
counter.fetch_add(1);     // 加法并返回旧值
counter.compare_exchange_strong(expected, desired);  // CAS 操作
```

**原子操作 vs 互斥锁**：

| 特性 | 原子操作 | 互斥锁 |
|-----|---------|-------|
| 适用场景 | 简单操作（计数器） | 复杂操作（多步骤） |
| 性能 | 更快 | 较慢 |
| 阻塞 | 不阻塞 | 可能阻塞 |
| 复杂度 | 只能操作单个变量 | 可以保护任意代码块 |

## 三、数字图像处理 (数据基础)

### 3.1 RGB 与 BGR，HWC 与 CHW

**RGB vs BGR**：

```
RGB: 红-绿-蓝 (OpenCV 以外的大多数库)
BGR: 蓝-绿-红 (OpenCV 默认)

像素值示例：
  RGB: [255, 0, 0] = 红色
  BGR: [0, 0, 255] = 红色（OpenCV 中）

转换：
  cv::cvtColor(image, image, cv::COLOR_BGR2RGB);
```

**HWC vs CHW**：

```
HWC (Height × Width × Channel)：OpenCV 默认
  内存布局: [像素1的BGR][像素2的BGR][像素3的BGR]...
  形状: (256, 256, 3)

CHW (Channel × Height × Width)：PyTorch 默认
  内存布局: [所有B通道][所有G通道][所有R通道]
  形状: (3, 256, 256)
```

**内存布局图解**：

```
HWC 布局 (OpenCV):
像素 (0,0)  像素 (0,1)  像素 (0,2) ...
[B G R]     [B G R]     [B G R]    ...
   ↓
内存顺序: B0 G0 R0 B1 G1 R1 B2 G2 R2 ...

CHW 布局 (PyTorch):
B通道:     G通道:     R通道:
[B0 B1 B2] [G0 G1 G2] [R0 R1 R2]
   ↓
内存顺序: B0 B1 B2 ... G0 G1 G2 ... R0 R1 R2 ...
```

**转换示例**：

```cpp
// OpenCV (HWC) → PyTorch (CHW)
cv::Mat image = cv::imread("input.png");  // HWC: (256, 256, 3)

// 方法1：手动转换
cv::Mat channels[3];
cv::split(image, channels);  // 分离通道
// channels[0] = B, channels[1] = G, channels[2] = R

// 方法2：使用 cv::dnn::blobFromImage
cv::Mat blob = cv::dnn::blobFromImage(image, 1.0/255.0, cv::Size(256, 256));
// 输出形状: (1, 3, 256, 256) - NCHW
```

```python
# PyTorch 中的转换
import torch
import numpy as np

# numpy HWC → PyTorch CHW
image_hwc = np.random.rand(256, 256, 3)  # HWC
image_chw = np.transpose(image_hwc, (2, 0, 1))  # CHW
tensor = torch.from_numpy(image_chw)  # (3, 256, 256)

# 或者使用 permute
tensor_hwc = torch.randn(256, 256, 3)
tensor_chw = tensor_hwc.permute(2, 0, 1)  # (3, 256, 256)
```

image_uint8 = torch.randint(0, 256, (3, 256, 256), dtype=torch.uint8)

std::vector<cv::Mat> splitImage(const cv::Mat& image, int tile_size, int overlap) {
    std::vector<cv::Mat> tiles;
    int step = tile_size - overlap;

    for (int y = 0; y < image.rows; y += step) {
        for (int x = 0; x < image.cols; x += step) {

    }
    return tiles;

### 4.1 卷积神经网络基础

**卷积层 (Convolution)**：

**常见网络模块组合**：

```python
# Conv + BN + ReLU：深度学习标准组件
class ConvBNReLU(nn.Module):
    def __init__(self, in_ch, out_ch, kernel_size=3, stride=1, padding=1):
        super().__init__()
        self.conv = nn.Conv2d(in_ch, out_ch, kernel_size, stride, padding, bias=False)
        self.bn = nn.BatchNorm2d(out_ch)
        self.relu = nn.ReLU(inplace=True)

    def forward(self, x):
        return self.relu(self.bn(self.conv(x)))
```

        high_path = os.path.join(self.high_dir, self.images[idx])

### 4.4 训练循环

**完整训练脚本模板**：

```python
import torch
import torch.nn as nn
import torch.optim as optim
from tqdm import tqdm

def train():
    # 1. 设备配置
    device = 'cuda' if torch.cuda.is_available() else 'cpu'

    # 2. 模型、损失函数、优化器
    model = UNetRepVGG().to(device)
    criterion = nn.L1Loss()
    optimizer = optim.Adam(model.parameters(), lr=1e-4)
    scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=200)

    # 3. 数据加载
    train_loader = DataLoader(train_dataset, batch_size=8, shuffle=True)
    val_loader = DataLoader(val_dataset, batch_size=1, shuffle=False)

    # 4. 训练循环
    best_psnr = 0
    for epoch in range(200):
        # 训练阶段
        model.train()
        train_loss = 0

        for low, high in tqdm(train_loader, desc=f'Epoch {epoch}'):
            low = low.to(device)
            high = high.to(device)

            # 前向传播
            pred = model(low)
            loss = criterion(pred, high)

            # 反向传播
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

            train_loss += loss.item()

        # 更新学习率
        scheduler.step()

        # 验证阶段
        model.eval()
        val_psnr = 0
        with torch.no_grad():
            for low, high in val_loader:
                pred = model(low.to(device))
                val_psnr += calculate_psnr(pred, high.to(device))

        val_psnr /= len(val_loader)
        print(f'Epoch {epoch}: Loss={train_loss:.4f}, PSNR={val_psnr:.2f}dB')

        # 保存最佳模型
        if val_psnr > best_psnr:
            best_psnr = val_psnr
            torch.save(model.state_dict(), 'best_model.pth')
```

### 4.6 图像质量评估指标

**PSNR (峰值信噪比)**：

```python
import torch

def calculate_psnr(pred, target, max_val=1.0):
    """
    计算 PSNR（越高越好）

    公式: PSNR = 10 * log10(MAX^2 / MSE)

    参数:
        pred: 预测图像 (B, C, H, W)，范围 [0, 1]
        target: 目标图像 (B, C, H, W)
        max_val: 像素最大值

    返回:
        PSNR 值 (dB)
    """
    mse = torch.mean((pred - target) ** 2)
    if mse == 0:
        return float('inf')
    psnr = 10 * torch.log10(max_val ** 2 / mse)
    return psnr.item()
```

**SSIM (结构相似性)**：

```python
from skimage.metrics import structural_similarity as ssim
import numpy as np

def calculate_ssim(pred, target):
    """
    计算 SSIM（越高越好，最大值 1）

    参数:
        pred: 预测图像 (H, W, C)，范围 [0, 255]
        target: 目标图像 (H, W, C)

    返回:
        SSIM 值
    """
    # 转换为 numpy 数组
    if isinstance(pred, torch.Tensor):
        pred = pred.cpu().numpy()
        target = target.cpu().numpy()

    # 如果是 CHW 格式，转换为 HWC
    if pred.ndim == 4:
        pred = pred.squeeze(0).transpose(1, 2, 0)
        target = target.squeeze(0).transpose(1, 2, 0)

    # 计算 SSIM
    return ssim(pred, target, channel_axis=2, data_range=255)
```

**指标参考**：

| 指标 | 差 | 一般 | 好 | 优秀 |
|-----|----|----|----|----|
| PSNR | < 20dB | 20-25dB | 25-30dB | > 30dB |
| SSIM | < 0.7 | 0.7-0.85 | 0.85-0.95 | > 0.95 |
